# i18n Design — Frontend Internationalization

> Date: 2026-03-09
> Status: Approved
> Approach: Option A — Pure Frontend JSON Translation

## Goal

Add Chinese (zh-TW) / English language switching to the Web UI.
No backend changes. Strategy names stay in English; descriptions and
parameter hints are translated.

## Architecture

```
frontend/
├── index.html          ← elements get data-i18n="key" attributes
└── locales/
    ├── en.json         ← ~180 keys (UI + strategy descriptions)
    └── zh-TW.json      ← Traditional Chinese translations
```

### i18n Engine (~40 lines, inline in index.html)

| Function | Purpose |
|----------|---------|
| `loadLocale(lang)` | Fetch `/locales/{lang}.json`, cache in memory |
| `applyTranslations()` | Scan all `[data-i18n]` elements, set `textContent` |
| `t(key, fallback)` | Return translated string for use in JS template literals |
| `setLang(lang)` | Save to `localStorage`, reload translations, re-apply |

### Language Detection Order

1. `localStorage.getItem('lang')`
2. `navigator.language` → map `zh-*` to `zh-TW`, else `en`
3. Fallback: `en`

### Language Switcher

Small toggle in sidebar header: `EN | 中`
Click switches instantly without page reload.

## Translation Scope

| Category | Count | Method |
|----------|-------|--------|
| UI labels, buttons, table headers | ~130 | `data-i18n` attribute on HTML elements |
| Dynamic status messages | ~15 | `t('key')` calls in JS |
| Strategy descriptions | 11 | `strategy.desc.{name}` keys |
| Parameter descriptions | ~30 | `param.desc.{key}` keys |
| Strategy names | 11 | **NOT translated** (industry standard) |
| Number/date formatting | — | **NOT changed** (universal format) |

## JSON Key Convention

```
nav.*          — navigation / mode toggle buttons
label.*        — form labels
btn.*          — action buttons
status.*       — dynamic status messages
metric.*       — performance metric labels
table.*        — table column headers
section.*      — section headings
empty.*        — empty state titles and descriptions
strategy.desc.* — strategy descriptions (keyed by strategy name)
param.desc.*   — parameter descriptions (keyed by param key)
```

## Implementation Notes

### HTML Markup Pattern

```html
<!-- Before -->
<label>Trading Pair</label>
<button>Run Backtest</button>

<!-- After -->
<label data-i18n="label.trading_pair">Trading Pair</label>
<button data-i18n="btn.run_backtest">Run Backtest</button>
```

English text stays as default content (fallback if JSON fails to load).

### JS Dynamic Text Pattern

```javascript
// Before
statusEl.innerHTML = '<div>Fetching data & running backtest...</div>';

// After
statusEl.innerHTML = `<div>${t('status.fetching', 'Fetching data & running backtest...')}</div>`;
```

### Strategy Description Rendering

```javascript
// In renderParams():
const descKey = `strategy.desc.${strategy.name}`;
document.getElementById('strategy-desc').textContent =
  t(descKey, strategy.description);  // fallback to API description
```

### Parameter Description Rendering

```javascript
// In renderParams():
const paramDescKey = `param.desc.${key}`;
div.innerHTML = `
  <label title="${t(paramDescKey, spec.description)}">${key}</label>
  ...
`;
```

## Out of Scope (YAGNI)

- No plural form handling (Chinese doesn't need it)
- No date/number locale formatting
- No backend code changes
- No RTL support
- No translation management UI
- No build step or tooling
